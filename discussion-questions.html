You can only traverse a singly-linked list in one direction (it only has a next attribute), and you can traverse a doubly-linked list in both directions (it has a next and a previous attributes). 3) Why is it faster to append to a linked list if we keep track of the tail as an attribute? If you don't keep track of the tail, to find the end of a linked list, you will have to traverse the entire list by looking at each node's next attribute until the next attribute is None, then add append the node to the end. The runtime for this is O(n). If you keep track of the tail as an attribute, you can simply find the tail (which has a runtime of O(1)) and append to the tail.
